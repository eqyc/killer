# 安全策略 (Security Policy)

我们非常重视 KILLER ERP 系统的安全性。作为一套处理敏感企业数据的系统，我们致力于保护用户数据并及时响应安全威胁。

## 1. 报告漏洞 (Reporting a Vulnerability)

**请勿使用公开的 GitHub Issue 报告安全漏洞。**

如果您发现任何安全漏洞，请通过以下方式私下告知我们：
*   **邮件**: [security@eqy.cc](mailto:security@eqy.cc)
*   **响应时间**: 我们通常会在 48 小时内确认收到报告，并提供初步的修复计划。

我们鼓励负责任的披露（Responsible Disclosure）。在漏洞修复并发布更新之前，请给予我们足够的时间来处理，不要公开细节。

## 2. 受支持的版本 (Supported Versions)

目前项目处于快速迭代阶段，我们仅对以下版本提供安全更新：

| 版本 | 是否受支持 |
| :--- | :--- |
| main 分支 | ✅ 当前开发版 |
| < v0.1.0 | ❌ 不再支持 |

## 3. 安全设计准则 (Security Design)

KILLER ERP 在架构层面集成了以下安全实践：

*   **内存安全**: 利用 Rust 的所有权模型，在编译期杜绝缓冲区溢出和悬垂指针等常见漏洞。
*   **依赖扫描**: 通过 `cargo audit` 和 GitHub Actions 自动检查已知漏洞的依赖库。
*   **最小权限原则**: 内部服务通过 mTLS 通信，并使用 HashiCorp Vault 管理所有 Secrets。
*   **身份验证**: 强制使用 JWT (RS256) 和 OIDC 协议。
*   **审计追踪**: 所有数据变更均记录在不可篡改的事件日志中。

## 4. 修复与致谢 (Remediation & Acknowledgement)

对于确认的安全漏洞，我们将：
1.  在私有环境中修复。
2.  向受影响的用户发出预警（如果适用）。
3.  发布修复补丁并更新版本号。
4.  在安全公告中向发现者致谢（除非您要求匿名）。

感谢您帮助我们构建更安全的 KILLER ERP 系统！
